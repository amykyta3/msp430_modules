Configuration Defines

Configuration defines for the Timer module

The timer module uses the MSP430's hardware timer. Since it only uses Capture-Control block 0, it can share the same hardware timer device with the following other modules:

  • Pushbutton Events (Only uses Capture-Control blocks 1 and 2)

To ensure proper operation when sharing the timer, All of the timer settings must be identical. Other signals using the same IO port cannot use interrupts outside of this module.

void timer_start ( timer_t *  timerid,
struct timerctl *  settings 
)

Creates a new or resumes an existing interval timer.

The new timer object is returned in the buffer pointed to by timerid, which must be a non-NULL pointer. This timer object can not be deallocated until after the timer has been stopped.

The settings argument points to a timerctl structure that specifies how the timer operates. A prevoiously stopped timer can be resumed by passing a NULL pointer into the settings argument.

Parameters
timeridPointer to the timer object
settingsPointer to a timerctl struct which defines the behavior of a new timer. A NULL pointer will resume a previously stopped timer defined by timerid

References timerctl::interval_ms, and timer_stop().

◆ timer_stop()

void timer_stop ( timer_t *  timerid)

Stops a currently running timer.

When stopping a timer, the state of the timer count is preserved allowing it to be resumed at a later time. A stopped timer can be resumed by passing it into timer_start() along with a NULL pointer in place of the settings argument

Parameters
timeridPointer to the timer object to stop
