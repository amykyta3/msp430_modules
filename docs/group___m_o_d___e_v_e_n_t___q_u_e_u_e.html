Detailed Description

A simple first-in first-out event handler.

Author
Alex Mykyta

Template main.c using the Event Queue

#include <msp430_xc.h>
#include <event_queue.h>
int main(void){
WDTCTL = WDTPW + WDTHOLD; // Stop the Watchdog Timer
__disable_interrupt(); // Disable Interrupts
// Initialize Port IO
...
// Initialize the system's clocks
...
// Initialize the event handler
event_init();
// Initialize any modules that generate events
...
// Turn on interrupts
__enable_interrupt();
// Start the event handler
event_StartHandler();
// Program never reaches this
return(0);
}
void onIdle(void){
// onIdle is called when no events occur.
// To save power, the CPU can be put into a sleep state here.
}

Schedule a function to be called in the event queue along with related data.

Parameters
[in]fptrPointer to the function to be called
[in]eventDataPointer to the data to be pushed into the queue (If not used, enter NULL)
[in]sizeNumber of bytes to be pushed (if none required, use size of 0)
Return values
RES_OKEvent added successfully
RES_FULLNot enough room in the event queue. Event was not added.

The function to be called must not take any input parameters nor can it return any values. If pusing additional data with the event, the event called MUST have a matching event_PopEventData(). Every additional byte pushed into the event queue MUST be popped out regardless if it is used or not.

References fifo_wrcount(), fifo_write(), RES_FULL, and RES_OK.

◆ event_YieldEvent()

void event_YieldEvent ( void  )

Yields execution of the current event to the next pending event in the queue.

Calling this function allows the next event in the queue to be executed. If no events are in the queue or the next event is already active, the onIdle() event is processed.

event_YieldEvent() can be called occasionally when performing a time-consuming operation within an event such as a polling loop. Doing so allows other events that may have piled up in the meantime to be processed.
