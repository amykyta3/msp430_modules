◆ UIO_RX_MODE

#define UIO_RX_MODE   0

0 = Polling Mode : No buffers used. RX is done directly by polling within function calls.
1 = Interrupt Mode : RX triggers an interrupt and data is pushed into a FIFO.
2 = DMA Mode : RX data is automatically loaded into a buffer using DMA.

◆ UIO_TX_MODE

#define UIO_TX_MODE   0

0 = Polling Mode : No buffers used. TX functions will block until data is transmitted.
1 = Interrupt Mode : TX data is buffered. TX operations are handled using interrupts.

◆ uart_gets_s()

char * uart_gets_s ( char *  str,
size_t  n 
)

Reads in a string of characters until a new-line character ( \n) is received.

  • Reads at most n-1 characters from the UART
  • Resulting string is null-terminated
  • If n is zero, a null character is written to str[0], and the function reads and discards characters until a new-line character is received.
  • If n-1 characters have been read, the function continues reading and discarding characters until a new-line character is received.
  • If an entire line is not immediately available, the function will block until it receives a new-line character.
Parameters
[out]strPointer to the destination string buffer
nThe size of the string buffer str
Returns
str on success. NULL otherwise

◆ uart_rx_dma_isr()

void uart_rx_dma_isr ( void  )

UART RX DMA Interrupt Service Routine.

If the UART RX mode is set to use DMA, the user must implement the DMA controller's ISR function. The ISR must call this function if the interrupt is for the RX DMA channel. This can be done using the is_uart_rx_dma_isr() function as follows:

if(is_uart_rx_dma_isr()){
uart_rx_dma_isr();
}
